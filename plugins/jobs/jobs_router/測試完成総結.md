## 修正された問題

### テスト実行時のモック問題（2026-03-11修正）

**問題**: pytestでモックが正しく動作せず、以下のエラーが発生
- モック関数が呼ばれない（call_count=0）
- 404エラー（ルートの二重プレフィックス）
- 暗号化テストの失敗（crypto関数がモックされていない）

**根本原因**:
1. Pythonのインポートメカニズムにより、`router.py`が`from .status_manager import get_job_result_sync`で関数をインポートする際、関数参照が固定される
2. 後から`status_manager.get_job_result_sync`をモックしても、routerモジュールは既に古い参照を持っている
3. ルーターが`prefix="/jobs"`を持つのに、`app.include_router(router, prefix="/jobs")`で二重にプレフィックスを追加していた

**解決方法**:
1. **ルーターモジュールの強制リロード**: `sys.modules`から削除して再インポート
2. **直接ネームスペースにパッチ**: `router_module.get_job_result_sync = mock`で直接代入
3. **暗号化関数のモック追加**: `verify_auth_hash`, `decrypt_opensearch_dashboard_payload`, `_get_shared_secret`を追加
4. **プレフィックス修正**: `app.include_router(router)`でプレフィックスを追加しない

詳細は `FIX_SUMMARY.md` を参照。

**修正前**: 3 failed, 30 passed, 3 skipped  
**修正後**: ✅ **33 passed, 3 skipped**
