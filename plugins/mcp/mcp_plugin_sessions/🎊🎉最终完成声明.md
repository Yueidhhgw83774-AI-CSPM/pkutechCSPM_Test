## 🔥 RecursionError 终极解决方案

### 问题演变

```python
# 第三轮：只限制了 pool
mock_pool = MagicMock(spec=['connection'])
mock_pool.connection = MagicMock(return_value=mock_conn)
# 问题：mock_pool.connection.xxx.yyy... 仍然可以无限调用

# 第四轮A：限制了 pool.connection 的返回值
mock_pool.connection = MagicMock(return_value=mock_conn, spec=['return_value'])
# 结果：部分测试通过，但 delete 操作仍然失败

# 第四轮B（终极版）：完整的三层 spec
mock_cursor = MagicMock(spec=['execute', 'fetchone', 'fetchall', ...])  # ← 第一层
mock_conn = MagicMock(spec=['cursor', '__aenter__', '__aexit__'])       # ← 第二层
mock_pool = MagicMock(spec=['connection'])                               # ← 第三层
mock_pool.connection = MagicMock(return_value=mock_conn, spec=['return_value'])  # ← 关键！
# 结果：✅ 彻底解决！
```

### 为什么这次成功了？

**完整的 spec 层级**:
1. ✅ Cursor 有 spec → 限制数据库操作方法
2. ✅ Connection 有 spec → 限制连接对象方法
3. ✅ Pool 有 spec → 限制连接池属性
4. ✅✅ **Pool.connection 也有 spec** → **防止无限链式调用**

这确保了：
- `mock_pool.connection()` ✅ 允许
- `mock_pool.connection().cursor()` ✅ 允许
- `mock_pool.connection().connection()` ❌ AttributeError（预期）
